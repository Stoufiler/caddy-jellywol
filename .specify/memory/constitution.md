# JellyWolProxy Constitution

## Core Principles

### I. Code Quality
Clean, maintainable, and well-documented code is essential for the long-term health of the project. All code should be written with clarity and simplicity in mind, and be easy for other developers to understand and modify.

### II. Testing Standards
Comprehensive and effective testing is non-negotiable. All new features and bug fixes must be accompanied by appropriate tests to ensure correctness and prevent regressions. The TDD (Test-Driven Development) methodology is strongly encouraged.

### III. User Experience Consistency
The user experience should be consistent and predictable across the entire application. This includes API design, command-line interfaces, and any other user-facing components.

### IV. Performance Requirements
The application must be efficient and responsive. Performance considerations should be a key part of the design and implementation process, and any performance-critical code should be benchmarked and optimized.

## Governance

This constitution supersedes all other practices. Amendments require documentation, approval, and a migration plan. All pull requests and reviews must verify compliance with these principles.
